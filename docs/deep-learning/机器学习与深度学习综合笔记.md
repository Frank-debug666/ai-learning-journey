> 阅读顺序：先看本笔记建立全局地图，再按文末索引进入专题笔记查细节。

## 0. 一张地图看懂 AI、机器学习与深度学习

```text
人工智能 AI
└─ 机器学习 ML：让机器从数据中学习规律
   ├─ 传统机器学习：KNN、线性模型、决策树、集成学习、K-Means
   └─ 深度学习 DL：使用多层神经网络自动学习特征
      ├─ MLP：表格与基础任务
      ├─ CNN：图像与空间结构
      ├─ RNN / LSTM / GRU：序列与时间关系
      └─ Transformer：注意力、大语言模型与多模态模型
```

机器学习和深度学习不是互相取代的关系：

| 情况 | 通常优先考虑 |
|---|---|
| 表格数据、样本较少、需要解释 | 传统机器学习 |
| 图片、文本、语音等非结构化数据 | 深度学习 |
| 数据量大、关系复杂、追求更高上限 | 深度学习 |
| 需要快速建立可靠 baseline | 简单机器学习模型 |

**核心认识：先理解任务和数据，再选择模型；复杂模型不一定更适合。**

## 1. 所有建模项目共用的一条主线

```text
定义问题与指标
→ 收集并理解数据
→ 划分训练集、验证集、测试集
→ 清洗数据与特征工程
→ 建立简单 baseline
→ 训练并验证模型
→ 选择最优方案
→ 在测试集做最终评估
→ 部署、监控与迭代
```

### 1.1 三个数据集的分工

| 数据集 | 通俗理解 | 用途 |
|---|---|---|
| 训练集 | 练习题 | 学习模型参数 |
| 验证集 | 模拟考试 | 调超参数、选模型、Early Stopping |
| 测试集 | 最终考试 | 模型确定后做一次公正评价 |

测试集不能被反复用于调参，否则它会逐渐变成另一份训练集。

### 1.2 什么是数据泄漏

数据泄漏是模型训练时使用了现实预测阶段不可能获得的信息。

常见例子：

- 测试样本混入训练集；
- 在划分数据前，用全部数据计算标准化统计量；
- 预测未来时使用了未来信息；
- 同一个用户或同一对象的高度相似记录跨越训练集与测试集。

**正确做法：预处理器只在训练集上 `fit`，验证集和测试集只做 `transform`。**

## 2. 先判断任务类型，再谈算法

### 2.1 监督学习

监督学习的数据带有标准答案。

| 任务 | 标签特点 | 例子 |
|---|---|---|
| 分类 | 离散类别 | 是否违约、图片属于哪一类 |
| 回归 | 连续数值 | 房价、销量、温度 |

### 2.2 无监督学习

无监督学习没有标准答案，目标是发现数据内部结构。

| 任务 | 例子 |
|---|---|
| 聚类 | 用户分群、商品分组 |
| 降维 | PCA、可视化高维数据 |
| 异常检测 | 找到少见或异常样本 |

### 2.3 其他学习方式

- 半监督学习：少量有标签数据 + 大量无标签数据；
- 自监督学习：从数据本身构造学习目标，例如预测被遮住的词；
- 强化学习：智能体与环境交互，通过奖励学习决策策略。

## 3. 机器学习地基：数据与特征工程

传统机器学习通常依赖人工构造和处理特征；深度学习虽然能自动学习特征，但仍依赖可靠的数据处理。

### 3.1 常见数据处理

| 问题 | 常见方法 |
|---|---|
| 缺失值 | 删除、均值/中位数填充、模型填充 |
| 异常值 | 检查来源、截断、变换、RobustScaler |
| 重复数据 | 去重，检查是否造成数据泄漏 |
| 类别特征 | One-Hot、Ordinal Encoding、Embedding |
| 数值尺度不同 | 标准化、归一化 |
| 类别不均衡 | 分层抽样、类别权重、重采样、合理指标 |

### 3.2 标准化与归一化

**标准化 Standardization**

```text
x' = (x - 均值) / 标准差
```

- 常见结果：均值约为 0，标准差约为 1；
- 不会自动把任意分布变成正态分布；
- 均值和标准差仍会受到异常值影响。

**归一化 Min-Max Scaling**

```text
x' = (x - 最小值) / (最大值 - 最小值)
```

- 通常把训练集范围映射到 `[0, 1]`；
- 对异常值较敏感；
- 新数据可能超出 `[0, 1]`。

对距离和梯度敏感的模型通常需要缩放，例如 KNN、SVM、线性模型和神经网络。决策树、随机森林等树模型通常不需要。

### 3.3 编码类别特征

| 方法 | 适用 |
|---|---|
| One-Hot | 无序且类别数量不太多的特征 |
| Ordinal Encoding | 确实存在顺序的类别 |
| LabelEncoder | 常用于目标标签 `y`，不应随意用于无序输入特征 |
| Embedding | 高基数类别、文本 token、深度学习模型 |

### 3.4 特征选择与降维

- 特征选择：保留原始特征中的有用部分，可解释性较强；
- PCA：把原特征组合为新的主成分，保留主要方差；
- t-SNE / UMAP：主要用于探索性可视化，不宜默认作为正式预测流水线输入。

## 4. 传统机器学习核心算法

### 4.1 算法选择速查

| 算法 | 直觉 | 优点 | 注意点 |
|---|---|---|---|
| 线性回归 | 用直线或超平面预测连续值 | 简单、可解释、适合 baseline | 只能直接表达线性关系 |
| 逻辑回归 | 线性打分后映射为分类概率 | 快、可解释、强 baseline | 对特征尺度与特征表达敏感 |
| KNN | 看附近样本是什么 | 直观、几乎无需训练 | 预测慢、怕量纲差异和高维 |
| 决策树 | 连续提问并分支 | 易解释、无需缩放 | 单树容易过拟合 |
| 随机森林 | 多棵树投票 | 稳定、通用、抗过拟合 | 模型较大，解释弱于单树 |
| Boosting | 后续模型持续纠正错误 | 表格任务常很强 | 参数较多，需防过拟合 |
| K-Means | 围绕 K 个中心自动分组 | 简单、快速 | 需要预设 K，怕尺度和异常值 |

### 4.2 KNN 与 K-Means 不要混淆

```text
KNN：有标签，找 K 个邻居做分类或回归
K-Means：无标签，学习 K 个聚类中心做分组
```

### 4.3 Bagging 与 Boosting

```text
Bagging：多个模型相对独立地训练，再投票或平均
代表：随机森林

Boosting：模型串行训练，后面的模型重点修正已有错误
代表：AdaBoost、Gradient Boosting、XGBoost、LightGBM
```

“Boosting 都是在拟合残差”并不完全准确：Gradient Boosting 拟合损失的负梯度，在平方误差下才可直观理解为拟合残差。

## 5. 模型评估：分数必须和问题匹配

### 5.1 二分类混淆矩阵

| | 预测正类 | 预测负类 |
|---|---:|---:|
| 实际正类 | TP | FN |
| 实际负类 | FP | TN |

`T/F` 表示预测是否正确，`P/N` 表示模型预测为正类或负类。

### 5.2 分类指标

```text
Accuracy  = (TP + TN) / 全部样本
Precision = TP / (TP + FP)
Recall    = TP / (TP + FN)
F1        = Precision 与 Recall 的调和平均
```

| 业务重点 | 更应关注 |
|---|---|
| 整体正确率，类别较均衡 | Accuracy |
| 误报代价高 | Precision |
| 漏报代价高 | Recall |
| 类别不均衡且需兼顾查准查全 | F1、PR-AUC |
| 比较不同阈值下的排序能力 | ROC-AUC / PR-AUC |

医疗场景不能笼统说“关注 Precision”或“关注 Recall”，必须根据具体错误代价判断。例如疾病筛查通常更怕漏诊，因此更重视 Recall。

### 5.3 回归指标

| 指标 | 特点 |
|---|---|
| MAE | 与目标同单位，对大误差的惩罚较温和 |
| MSE | 对大误差惩罚更强，单位被平方 |
| RMSE | 对大误差敏感，且恢复到目标单位 |
| R² | 衡量相对基准模型解释了多少方差，可能为负 |

### 5.4 不要只看一个分数

还应检查：

- 训练集与验证集之间是否存在明显差距；
- 不同类别、不同人群和不同时间段的表现；
- 混淆矩阵与错误样本；
- 推理速度、资源占用与业务成本。

## 6. 拟合、泛化与可靠验证

| 状态 | 表现 | 常见处理 |
|---|---|---|
| 欠拟合 | 训练集和验证集都差 | 增加特征、提升模型容量、训练更久 |
| 合理拟合 | 训练集和验证集都较好，差距合理 | 继续验证稳定性 |
| 过拟合 | 训练集好，验证集明显变差 | 正则化、更多数据、简化模型、早停 |

常用验证方法：

- 固定训练/验证/测试划分；
- K 折交叉验证；
- 分类任务使用分层抽样；
- 时间序列必须按时间划分，不能随机打乱未来和过去。

## 7. 从机器学习进入深度学习

传统机器学习常见流程：

```text
人工设计特征 → 模型学习特征与目标的关系
```

深度学习常见流程：

```text
原始或轻度处理的数据 → 神经网络自动学习多层特征 → 完成任务
```

神经网络的基本单元：

```text
输入 x
→ 线性变换 z = Wx + b
→ 激活函数 a = f(z)
→ 多层堆叠
→ 输出预测
```

激活函数让多层网络能够表达非线性关系。没有非线性激活，多层线性层仍等价于一个线性变换。

## 8. PyTorch 最重要的三份数据契约

### 8.1 Shape：包装规格

| 符号 | 含义 |
|---|---|
| `B` | Batch Size |
| `F` | 特征数 |
| `C` | 通道数或类别数 |
| `H, W` | 图像高度和宽度 |
| `T` | 序列长度 |
| `D` | Embedding / Hidden 维度 |
| `V` | 词表大小 |

常见 Shape：

```text
表格：[B, F]
图像：[B, C, H, W]
token id：[B, T]
词向量：[B, T, D]
多分类 logits：[B, C]
序列生成 logits：[B, T, V]
```

### 8.2 dtype：数据类型

| 数据 | 常用 dtype |
|---|---|
| 输入特征、回归标签 | `torch.float32` |
| 二分类、多标签标签 | `torch.float32` |
| 多分类类别索引 | `torch.long` |
| Embedding 的 token id | `torch.long` |
| 布尔掩码 | `torch.bool` |

### 8.3 device：存放位置

模型、输入和标签必须位于同一设备：

```python
model = model.to(device)
x = x.to(device)
y = y.to(device)
```

## 9. 任务决定输出层与损失函数

| 任务 | 模型输出 | 标签 | 推荐损失 |
|---|---|---|---|
| 回归 | 与标签同 Shape 的浮点数 | 同 Shape float | `MSELoss` / `SmoothL1Loss` |
| 二分类 | `[B]` 或 `[B,1]` logits | 同 Shape float 0/1 | `BCEWithLogitsLoss` |
| 多分类 | `[B,C]` logits | `[B]` long 类别索引 | `CrossEntropyLoss` |
| 多标签 | `[B,C]` logits | `[B,C]` float 0/1 | `BCEWithLogitsLoss` |
| 序列生成 | `[B,T,V]` logits | `[B,T]` long | `CrossEntropyLoss` |

### 最重要的避坑规则

```python
# 多分类：不要先 Softmax
loss = nn.CrossEntropyLoss()(logits, target)

# 二分类：不要先 Sigmoid
loss = nn.BCEWithLogitsLoss()(logits, target)
```

`CrossEntropyLoss` 支持类别索引标签，也支持符合要求的类别概率目标；入门分类任务优先使用 `[B]` 的 `long` 类别索引。

## 10. 深度学习训练闭环

```python
for x, y in train_loader:
    x, y = x.to(device), y.to(device)

    optimizer.zero_grad()
    logits = model(x)
    loss = loss_fn(logits, y)
    loss.backward()
    optimizer.step()
```

通俗理解：

```text
清空旧梯度
→ 模型先答题
→ 损失函数判卷
→ 反向传播分析每个参数的责任
→ 优化器修改参数
```

### batch、iteration、epoch

- batch：一次交给模型的一组样本；
- iteration：处理一个 batch，通常更新一次参数；
- epoch：完整遍历一次训练集。

### 训练、验证、推理

| 阶段 | 模式 | 梯度 | 参数更新 |
|---|---|---|---|
| 训练 | `model.train()` | 开启 | 是 |
| 验证/测试 | `model.eval()` | `torch.no_grad()` | 否 |
| 推理 | `model.eval()` | `torch.no_grad()` | 否 |

`model.eval()` 与 `torch.no_grad()` 作用不同，评估和推理时通常都需要。

## 11. 优化器、学习率与正则化

### 11.1 优化器

| 场景 | 常用起点 |
|---|---|
| 快速 baseline | Adam |
| Transformer、大模型微调 | AdamW |
| 经典 CNN，愿意认真调参 | SGD + Momentum |
| 稀疏特征 | Adagrad |

不存在永远最好的优化器。模型、数据、学习率和训练预算会共同影响结果。

### 11.2 学习率

```text
太大：Loss 震荡、发散、NaN
太小：下降缓慢，训练很久仍不收敛
```

常见调度器：

- `StepLR`：固定间隔衰减；
- `MultiStepLR`：在指定里程碑衰减；
- `ExponentialLR`：指数衰减；
- Cosine Annealing：余弦衰减；
- Warmup：训练前期逐渐增大学习率。

### 11.3 正则化

| 方法 | 核心作用 |
|---|---|
| Weight Decay | 限制权重过大 |
| Dropout | 训练时随机屏蔽部分神经元 |
| 数据增强 | 增加有效数据多样性 |
| Early Stopping | 验证集不再改善时停止 |
| Label Smoothing | 减少分类模型过度自信 |

BatchNorm 可能带来一定正则化效果，但其主要目标是改善优化与训练稳定性，不能简单等同于正则化层。

## 12. CNN：让模型理解图像空间

图像在磁盘或数组中常是 `[H,W,C]`，PyTorch CNN 通常使用：

```text
[B, C, H, W]
```

典型流程：

```text
图片
→ 卷积：提取局部特征
→ 激活：加入非线性
→ 池化/步幅卷积：压缩空间尺寸
→ 更深层卷积：学习更抽象特征
→ 分类头：输出类别 logits
```

卷积输出尺寸：

```text
H_out = floor((H + 2P - D(K-1) - 1) / S) + 1
```

其中 `K` 是卷积核大小，`S` 是步幅，`P` 是填充，`D` 是 dilation。

**关键直觉：浅层常学习边缘和纹理，深层逐步组合为部件和语义。**

## 13. RNN、LSTM 与 GRU：让模型记住顺序

RNN 按时间步处理序列：

```text
当前输入 x_t + 上一步隐藏状态 h_(t-1)
→ 当前隐藏状态 h_t
```

PyTorch RNN 默认 Shape：

```text
输入 x：       [T, B, input_size]
初始状态 h0：  [layers, B, hidden_size]
输出 output：  [T, B, hidden_size]
最终状态 hn：  [layers, B, hidden_size]
```

使用 `batch_first=True` 后，输入和输出主 Shape 变为 `[B,T,D]`，更容易与 DataLoader 和 Embedding 配合。

| 模型 | 特点 |
|---|---|
| RNN | 结构简单，但长序列容易梯度消失或爆炸 |
| LSTM | 使用输入门、遗忘门、输出门和细胞状态 |
| GRU | 门控更精简，参数通常少于 LSTM |

RNN 生成任务常见 Shape：

```text
token id [B,T]
→ Embedding [B,T,E]
→ RNN [B,T,H]
→ Linear [B,T,V]
→ 与目标 token 计算 CrossEntropyLoss
```

## 14. Transformer：用注意力建立全局联系

Transformer 不依赖循环逐步传递信息，而是让每个位置直接关注其他位置。

### 14.1 自注意力

```text
Q：我想找什么
K：我有什么标签
V：我实际携带的信息
```

公式：

```text
Attention(Q,K,V) = softmax(QKᵀ / √d_k) V
```

步骤：

1. `QKᵀ` 计算位置之间的相关分数；
2. 除以 `√d_k` 避免分数过大导致 Softmax 饱和；
3. Softmax 得到注意力权重；
4. 用权重对 `V` 做加权求和。

### 14.2 多头注意力与位置

- 多头注意力让模型在不同表示子空间学习不同关系；
- 自注意力本身不编码顺序，因此必须加入位置编码或位置机制；
- Decoder 生成时使用因果 Mask，避免看到未来 token。

### 14.3 Transformer 家族

| 结构 | 代表 | 擅长 |
|---|---|---|
| Encoder-only | BERT | 理解、分类、抽取 |
| Decoder-only | GPT、LLaMA | 自回归生成 |
| Encoder-Decoder | T5、BART | 翻译、摘要、序列到序列 |

普通全注意力对序列长度的计算和内存开销通常为二次增长，这是长上下文建模的重要瓶颈。

## 15. Debug：模型不工作时按顺序检查

```text
1. 任务定义和指标是否正确？
2. 数据划分是否泄漏？
3. 打印一个 batch 的 shape、dtype、device 和数值范围
4. 打印 logits.shape，检查模型最后输出
5. 核对 Loss 的输入、标签 Shape 与 dtype
6. 检查 zero_grad → forward → loss → backward → step
7. 用极小数据集尝试过拟合
8. 检查梯度、NaN / Inf 与学习率
9. 再考虑换模型或增加复杂技巧
```

常见报错翻译：

| 报错 | 常见原因 |
|---|---|
| `mat1 and mat2 shapes cannot be multiplied` | Linear 输入最后一维不匹配 |
| `Expected ... Long` | CrossEntropyLoss 标签或 Embedding 输入不是 long |
| `Target size must be the same as input size` | BCE/回归的预测与标签 Shape 不一致 |
| `Expected all tensors to be on the same device` | 模型、输入、标签不在同一设备 |

## 16. 本轮统一纠错

以下是原分散笔记中容易误记、现已统一修正的重点：

1. 混淆矩阵中，实际正类预测负类是 `FN`；实际负类预测负类是 `TN`。
2. Recall 公式是 `TP / (TP + FN)`，不是 `TP / (TP + TN)`。
3. `T/F` 表示预测是否正确，`P/N` 表示预测的正负类别。
4. 标准化不会自动把任意分布变为正态分布，也不能称为天然抗异常值。
5. BCE、MSE、MAE、SmoothL1 的目标不要求必须是二维，只需满足对应 Shape 和 dtype 契约。
6. MSE 会更重地惩罚大误差，但它本身不必然导致梯度爆炸。
7. 标准训练顺序是先 `optimizer.zero_grad()`，再 `loss.backward()`，最后 `optimizer.step()`。
8. 指数学习率衰减器是 `ExponentialLR`，不是 `MultiStepLR`。
9. RNN 输出最后一维是 `hidden_size`，不是 Embedding 的词向量维度。
10. 不存在公认的“模型超过 10B 参数才会出现涌现能力”的固定阈值。

## 17. 学习优先级

### 第一阶段：必须熟练

- 监督学习、无监督学习、分类、回归；
- 数据集划分、数据泄漏、特征预处理；
- 线性回归、逻辑回归、KNN、决策树、随机森林；
- Accuracy、Precision、Recall、F1、MAE、RMSE；
- Shape、dtype、device；
- Loss 契约与 PyTorch 训练循环。

### 第二阶段：形成模型直觉

- 过拟合、正则化、交叉验证；
- CNN 的卷积与 Shape；
- Embedding、RNN/LSTM/GRU；
- Self-Attention 与 Transformer。

### 第三阶段：工程落地

- Pipeline、超参数搜索与实验记录；
- Checkpoint、Early Stopping 与可复现；
- 部署、延迟、监控、数据漂移；
- 大模型微调、RAG、评测与应用工程。
