# 深度学习常见损失函数汇总

| 类别       | 损失函数                 | PyTorch                    | 适用场景                     | 核心思想                                 | 优点                   | 缺点               |
| ---------- | ------------------------ | -------------------------- | ---------------------------- | ---------------------------------------- | ---------------------- | ------------------ |
| 回归       | MSELoss                  | `nn.MSELoss()`             | 房价预测、销量预测、温度预测 | 误差平方后求平均                         | 平滑、易求导、收敛稳定 | 对异常值敏感       |
| 回归       | L1Loss(MAE)              | `nn.L1Loss()`              | 存在异常值的数据             | 误差绝对值求平均                         | 抗异常值能力强         | 收敛速度较慢       |
| 回归       | SmoothL1Loss(Huber Loss) | `nn.SmoothL1Loss()`        | 目标检测、边界框回归         | 小误差用MSE，大误差用L1                  | 兼顾稳定性和鲁棒性     | 参数选择较复杂     |
| 回归       | LogCosh Loss             | 自定义                     | 高精度回归                   | 类似MSE但对异常值更友好                  | 平滑可导               | PyTorch无内置实现  |
| 二分类     | BCELoss                  | `nn.BCELoss()`             | 猫狗分类、垃圾邮件识别       | 二元交叉熵                               | 理论简单               | 数值不稳定         |
| 二分类     | BCEWithLogitsLoss        | `nn.BCEWithLogitsLoss()`   | 企业级二分类项目             | Sigmoid + BCELoss                        | 数值稳定、推荐使用     | 无明显缺点         |
| 多分类     | CrossEntropyLoss         | `nn.CrossEntropyLoss()`    | 图像分类、文本分类           | Softmax + 交叉熵                         | 最常用分类损失         | 需正确处理标签格式 |
| 多分类     | NLLLoss                  | `nn.NLLLoss()`             | 特殊分类网络                 | 负对数似然                               | 灵活                   | 需先计算LogSoftmax |
| 相似度学习 | Contrastive Loss         | 自定义                     | 人脸验证、图像检索           | 同类靠近、异类远离                       | 直观有效               | 样本构造复杂       |
| 相似度学习 | Triplet Loss             | `nn.TripletMarginLoss()`   | 人脸识别                     | Anchor-Positive接近，Anchor-Negative远离 | 效果优秀               | 训练较慢           |
| 相似度学习 | CosineEmbeddingLoss      | `nn.CosineEmbeddingLoss()` | 文本匹配、向量检索           | 最大化余弦相似度                         | 适合向量任务           | 对特征质量要求高   |
| 目标检测   | IoU Loss                 | 自定义                     | YOLO、Faster R-CNN           | 最大化预测框与真实框重叠率               | 符合检测目标           | 框不重叠时梯度消失 |
| 目标检测   | GIoU Loss                | 自定义                     | 目标检测                     | 改进IoU                                  | 解决无重叠问题         | 计算较复杂         |
| 目标检测   | DIoU Loss                | 自定义                     | YOLO系列                     | 增加中心点距离约束                       | 收敛更快               | 实现复杂           |
| 目标检测   | CIoU Loss                | 自定义                     | YOLOv5/v7/v8                 | 增加宽高比约束                           | 当前主流检测损失       | 计算量略大         |

# 损失函数输入输出维度要求 Shape 要求

> 这一部分专门解决一个问题：**模型最后一层输出应该是什么 shape？标签 y 应该是什么 shape？loss 输出又是什么 shape？**
>
> 记号说明：
>
> - `N`：batch size，批次大小
> - `C`：类别数，class number
> - `D`：向量维度，embedding dimension
> - `H, W`：图像高度和宽度
> - `T`：序列长度，例如文本/RNN中的时间步
> - `V`：词表大小，vocabulary size
> - `*`：任意维度，例如 `[N]`、`[N, 1]`、`[N, C]`、`[N, C, H, W]`

---

## 1. 总表：常见损失函数的输入输出 shape

| 类别 | 损失函数 | 模型输出 `y_pred/input` shape | 真实标签 `y/target` shape | loss 输出 shape | 标签类型要求 | 最容易犯的错误 |
|---|---|---|---|---|---|---|
| 回归 | `nn.MSELoss()` | 和 `target` 完全相同，例如 `[N, 1]`、`[N]`、`[N, C]` | 和 `input` 完全相同 | 默认标量；`reduction='none'` 时和输入相同 | `float` | `y_pred=[N,1]`，但 `y=[N]`，虽然可能广播但含义容易错 |
| 回归 | `nn.L1Loss()` | 和 `target` 完全相同 | 和 `input` 完全相同 | 默认标量；`reduction='none'` 时和输入相同 | `float` | 把分类任务误用成 L1Loss |
| 回归 | `nn.SmoothL1Loss()` | 和 `target` 完全相同，常见 `[N,4]` 用于边界框坐标 | 和 `input` 完全相同 | 默认标量；`reduction='none'` 时和输入相同 | `float` | 预测框 `[N,4]`，真实框 `[N]` 或 `[N,1]` |
| 回归 | LogCosh Loss | 和 `target` 完全相同 | 和 `input` 完全相同 | 通常默认标量；自定义时可保留逐元素 loss | `float` | 自定义时忘记 `mean()` 或 `sum()` |
| 二分类 | `nn.BCELoss()` | 概率值，范围 `[0,1]`；shape 和 target 相同，例如 `[N,1]` 或 `[N]` | 和 input 完全相同 | 默认标量；`reduction='none'` 时和输入相同 | `float`，值为 0/1 | 模型输出没有先过 `Sigmoid` |
| 二分类 | `nn.BCEWithLogitsLoss()` | 原始 logits，不要过 Sigmoid；shape 和 target 相同，例如 `[N,1]`、`[N]`、`[N,C]` | 和 input 完全相同 | 默认标量；`reduction='none'` 时和输入相同 | `float`，值为 0/1 | 在模型最后又加了 `Sigmoid`，导致重复 Sigmoid |
| 多分类 | `nn.CrossEntropyLoss()` | logits，常见 `[N,C]`；图像分割为 `[N,C,H,W]`；序列任务常整理成 `[N*T,C]` | 类别索引：`[N]`；分割：`[N,H,W]`；序列：`[N*T]` | 默认标量；`reduction='none'` 时为 `[N]` 或 `[N,H,W]` 等 | 类别索引用 `long`，不是 one-hot | target 写成 `[N,C]` 的 one-hot；或者模型最后加了 Softmax |
| 多分类 | `nn.NLLLoss()` | log-probabilities，常见 `[N,C]`，必须先经过 `LogSoftmax(dim=1)` | 类别索引 `[N]`；高维任务 `[N,H,W]` | 默认标量；`reduction='none'` 时为 `[N]` 或 `[N,H,W]` | `long` | 忘记在前面加 `LogSoftmax` |
| 相似度学习 | Contrastive Loss | 常见两个向量 `x1,x2`：`[N,D]`；或者距离 `distance`：`[N]` | 标签 `[N]`，常用 0/1 或 1/-1，取决于自定义公式 | 通常默认标量；`reduction='none'` 时 `[N]` | `float` 或 `long`，看自定义实现 | 不同教程对正负样本标签定义相反 |
| 相似度学习 | `nn.TripletMarginLoss()` | `anchor=[N,D]`，`positive=[N,D]`，`negative=[N,D]` | 无单独 target | 默认标量；`reduction='none'` 时 `[N]` | 三个输入都是 `float` 向量 | anchor/positive/negative 的 shape 不一致 |
| 相似度学习 | `nn.CosineEmbeddingLoss()` | `input1=[N,D]`，`input2=[N,D]` | `target=[N]`，值为 `1` 或 `-1` | 默认标量；`reduction='none'` 时 `[N]` | target 通常为 `float`，值为 1/-1 | target 写成 `[N,1]` 或忘记传 target |
| 目标检测 | IoU Loss | 预测框 `[N,4]`，常见格式 `[x1,y1,x2,y2]` 或 `[cx,cy,w,h]` | 真实框 `[N,4]` | 通常默认标量；`reduction='none'` 时 `[N]` | `float` | 预测框格式和真实框格式不一致 |
| 目标检测 | GIoU Loss | 预测框 `[N,4]` | 真实框 `[N,4]` | 通常默认标量；`reduction='none'` 时 `[N]` | `float` | 坐标没有归一化，或者 x1>x2/y1>y2 |
| 目标检测 | DIoU Loss | 预测框 `[N,4]` | 真实框 `[N,4]` | 通常默认标量；`reduction='none'` 时 `[N]` | `float` | 中心点距离计算时框格式混乱 |
| 目标检测 | CIoU Loss | 预测框 `[N,4]` | 真实框 `[N,4]` | 通常默认标量；`reduction='none'` 时 `[N]` | `float` | 宽高比项计算时出现除零，需要加 `eps` |

## 2. 回归损失函数 shape 讲解

### 2.1 MSELoss

```python
loss_f = torch.nn.MSELoss()
loss = loss_f(y_pred, y_true)
```

要求：

```text
y_pred.shape == y_true.shape
```

例子 1：预测一个连续值。

```python
y_pred = torch.randn(32, 1)   # [N, 1]
y_true = torch.randn(32, 1)   # [N, 1]
loss = loss_f(y_pred, y_true) # 标量
```

例子 2：预测多个连续值。

```python
y_pred = torch.randn(32, 4)   # [N, 4]
y_true = torch.randn(32, 4)   # [N, 4]
loss = loss_f(y_pred, y_true) # 标量
```

容易错的地方：

```python
y_pred = torch.randn(32, 1)   # [N, 1]
y_true = torch.randn(32)      # [N]
```

这种情况 PyTorch 可能会发生 broadcasting，代码不一定立刻报错，但数学含义容易错。建议统一成：

```python
y_true = y_true.reshape(-1, 1)
```

---

### 2.2 L1Loss / SmoothL1Loss

这两个和 MSELoss 的 shape 要求基本一样：

```text
y_pred.shape == y_true.shape
```

边界框回归中常见：

```python
pred_box = torch.randn(32, 4)   # [N, 4]
true_box = torch.randn(32, 4)   # [N, 4]
loss = torch.nn.SmoothL1Loss()(pred_box, true_box)
```

其中 `[N, 4]` 通常表示：

```text
[x, y, w, h]
```

或者：

```text
[x1, y1, x2, y2]
```

关键是：**预测框和真实框必须使用同一种坐标格式。**

---

## 3. 二分类损失函数 shape 讲解

### 3.1 BCELoss

`BCELoss` 要求输入已经是概率，也就是模型输出要经过 `Sigmoid`。

```python
model = torch.nn.Sequential(
    torch.nn.Linear(10, 1),
    torch.nn.Sigmoid()
)

loss_f = torch.nn.BCELoss()
```

shape 要求：

```text
y_pred.shape == y_true.shape
```

常见写法：

```python
y_pred = torch.rand(32, 1)       # [N, 1]，概率，范围 0~1
y_true = torch.randint(0, 2, (32, 1)).float()  # [N, 1]
loss = loss_f(y_pred, y_true)
```

---

### 3.2 BCEWithLogitsLoss 推荐

企业项目中更推荐：

```python
loss_f = torch.nn.BCEWithLogitsLoss()
```

因为它内部已经包含：

```text
Sigmoid + BCELoss
```

所以模型最后一层不要写 Sigmoid：

```python
model = torch.nn.Linear(10, 1)
```

正确 shape：

```python
logits = torch.randn(32, 1)      # [N, 1]，原始分数，不是概率
y_true = torch.randint(0, 2, (32, 1)).float()  # [N, 1]
loss = loss_f(logits, y_true)
```

也可以写成一维：

```python
logits = torch.randn(32)         # [N]
y_true = torch.randint(0, 2, (32,)).float()    # [N]
loss = loss_f(logits, y_true)
```

但是不要混用：

```python
logits = torch.randn(32, 1)      # [N, 1]
y_true = torch.randint(0, 2, (32,)).float()    # [N]，不推荐
```

---

## 4. 多分类损失函数 shape 讲解

### 4.1 CrossEntropyLoss 最重要

`CrossEntropyLoss` 用于多分类。它内部已经包含：

```text
LogSoftmax + NLLLoss
```

所以模型最后一层不要写 `Softmax`。

---

### 普通图像分类 / 表格分类

假设：

```text
N = 32
C = 10
```

模型输出应该是：

```python
logits = torch.randn(32, 10)  # [N, C]
```

标签应该是：

```python
y = torch.randint(0, 10, (32,))  # [N]
y.dtype == torch.long
```

计算：

```python
loss_f = torch.nn.CrossEntropyLoss()
loss = loss_f(logits, y)
```

注意：`y` 不是 one-hot。

错误写法：

```python
y_wrong = torch.randn(32, 10)  # [N, C]，普通多分类不要这样写
```

正确写法：

```python
y = torch.tensor([0, 3, 2, 9])  # 每个样本一个类别编号
```

---

### 图像分割任务

假设：

```text
N = 4
C = 3
H = 256
W = 256
```

模型输出：

```python
logits = torch.randn(4, 3, 256, 256)  # [N, C, H, W]
```

真实标签：

```python
y = torch.randint(0, 3, (4, 256, 256))  # [N, H, W]
y.dtype == torch.long
```

重点：

```text
input  有 C 这一维：[N, C, H, W]
target 没有 C 这一维：[N, H, W]
```

---

### RNN / NLP 序列分类任务

你的 RNN 语言模型中常见输出：

```text
RNN output: [T, B, hidden_size]
Linear 后: [T, B, V]
```

为了喂给 `CrossEntropyLoss`，通常展平成：

```python
logits = logits.reshape(-1, V)  # [T*B, V]
target = target.reshape(-1)     # [T*B]
loss = loss_f(logits, target)
```

也就是：

```text
input:  [样本数, 类别数] = [T*B, V]
target: [样本数]       = [T*B]
```

这正好对应你之前 RNN 代码里面：

```python
output = self.out(output.reshape(-1, output.shape[-1]))
```

它输出的是：

```text
[T*B, unique_count]
```

所以 target 也必须整理成：

```text
[T*B]
```

---

### 4.2 NLLLoss

`NLLLoss` 和 `CrossEntropyLoss` 的 target shape 基本一样，但是 input 不一样。

`NLLLoss` 的输入必须是 log probability：

```python
log_probs = torch.nn.LogSoftmax(dim=1)(logits)
loss = torch.nn.NLLLoss()(log_probs, y)
```

shape：

```text
log_probs: [N, C]
y:         [N]
```

如果你不想自己写 `LogSoftmax`，直接用：

```python
torch.nn.CrossEntropyLoss()
```

更简单。

---

## 5. 相似度学习损失函数 shape 讲解

### 5.1 TripletMarginLoss

用于三元组学习：

```text
Anchor   原始样本
Positive 同类样本
Negative 不同类样本
```

shape 要求：

```python
anchor   = torch.randn(32, 128)  # [N, D]
positive = torch.randn(32, 128)  # [N, D]
negative = torch.randn(32, 128)  # [N, D]

loss_f = torch.nn.TripletMarginLoss()
loss = loss_f(anchor, positive, negative)
```

没有单独的 `target`，因为正负关系已经由输入顺序决定了：

```text
anchor 要靠近 positive
anchor 要远离 negative
```

---

### 5.2 CosineEmbeddingLoss

用于判断两个向量应该相似还是不相似。

shape 要求：

```python
input1 = torch.randn(32, 128)  # [N, D]
input2 = torch.randn(32, 128)  # [N, D]
target = torch.ones(32)        # [N]
```

其中：

```text
target =  1 表示两个向量应该相似
target = -1 表示两个向量应该不相似
```

计算：

```python
loss_f = torch.nn.CosineEmbeddingLoss()
loss = loss_f(input1, input2, target)
```

注意：target 通常是 `[N]`，不是 `[N,1]`。

---

## 6. 目标检测损失函数 shape 讲解

### 6.1 SmoothL1Loss 用于边界框回归

Faster R-CNN 一类模型里，经常用 SmoothL1Loss 回归框坐标。

```python
pred_box = torch.randn(32, 4)  # [N, 4]
true_box = torch.randn(32, 4)  # [N, 4]
loss = torch.nn.SmoothL1Loss()(pred_box, true_box)
```

`4` 通常表示：

```text
[x1, y1, x2, y2]
```

或者：

```text
[cx, cy, w, h]
```

但是预测和标签必须统一格式。

---

### 6.2 IoU / GIoU / DIoU / CIoU Loss

这些通常是自定义实现，但常见 shape 是：

```python
pred_box = torch.randn(32, 4)  # [N, 4]
true_box = torch.randn(32, 4)  # [N, 4]
```

输出通常是：

```text
reduction='none': [N]
默认 mean:        标量
```

常见公式形式：

```text
IoU Loss = 1 - IoU
```

所以如果每个框算一个 IoU：

```python
iou = torch.rand(32)      # [N]
loss = (1 - iou).mean()   # 标量
```

---

## 7. 最容易混淆的 3 个 shape 对比

### 7.1 二分类 BCEWithLogitsLoss

```text
input:  [N, 1] 或 [N]
target: [N, 1] 或 [N]
要求：两者 shape 完全一致
```

例子：

```python
logits = torch.randn(32, 1)
y = torch.randint(0, 2, (32, 1)).float()
```

---

### 7.2 多分类 CrossEntropyLoss

```text
input:  [N, C]
target: [N]
```

例子：

```python
logits = torch.randn(32, 10)
y = torch.randint(0, 10, (32,)).long()
```

核心区别：

```text
BCE: input 和 target shape 一样
CrossEntropy: input 比 target 多一个类别维度 C
```

---

### 7.3 回归 MSELoss

```text
input:  [N, 1]
target: [N, 1]
```

或者：

```text
input:  [N]
target: [N]
```

要求：

```text
两者 shape 最好完全一致
```

## 8. 一张表记住核心区别

| 任务 | 最后一层输出 | target | 推荐 loss |
|---|---|---|---|
| 回归 | `[N, 1]` | `[N, 1]`，float | `MSELoss` |
| 二分类 | `[N, 1]` logits | `[N, 1]`，float，0/1 | `BCEWithLogitsLoss` |
| 多分类 | `[N, C]` logits | `[N]`，long，类别编号 | `CrossEntropyLoss` |
| 图像分割 | `[N, C, H, W]` logits | `[N, H, W]`，long | `CrossEntropyLoss` |
| RNN词预测 | `[T*B, V]` logits | `[T*B]`，long | `CrossEntropyLoss` |
| 向量相似 | `[N, D]` 和 `[N, D]` | `[N]`，1/-1 | `CosineEmbeddingLoss` |
| 三元组学习 | 三个 `[N, D]` | 不需要 target | `TripletMarginLoss` |
| 框回归 | `[N, 4]` | `[N, 4]` | `SmoothL1Loss` / `CIoU Loss` |

## 9. 实战检查清单

训练前可以先打印：

```python
print('y_pred shape:', y_pred.shape)
print('y_true shape:', y_true.shape)
print('y_pred dtype:', y_pred.dtype)
print('y_true dtype:', y_true.dtype)
```

然后按下面检查：

```text
回归任务：
    y_pred.shape == y_true.shape
    y_true.dtype 是 float

二分类任务：
    BCEWithLogitsLoss:
        y_pred.shape == y_true.shape
        y_pred 是 logits
        y_true 是 float，值为 0/1

多分类任务：
    CrossEntropyLoss:
        y_pred.shape == [N, C]
        y_true.shape == [N]
        y_true.dtype 是 long
        模型最后不要 Softmax

图像分割：
    y_pred.shape == [N, C, H, W]
    y_true.shape == [N, H, W]
    y_true.dtype 是 long

RNN 词预测：
    y_pred.shape == [T*B, V]
    y_true.shape == [T*B]
    y_true.dtype 是 long
```

## 10. 常见报错对应原因

| 报错/现象 | 常见原因 | 修改方法 |
|---|---|---|
| `Expected target size ...` | `CrossEntropyLoss` 的 input/target 维度不匹配 | 确认 input 是 `[N,C]`，target 是 `[N]` |
| `expected scalar type Long but found Float` | 多分类标签用了 float | `y = y.long()` |
| `result type Float can't be cast to Long` | 回归或 BCE 标签 dtype 不对 | `y = y.float()` |
| loss 不报错但训练异常 | `[N,1]` 和 `[N]` 发生了错误 broadcasting | 统一 reshape 成相同 shape |
| 二分类 loss 很奇怪 | `BCEWithLogitsLoss` 前面又手动加了 Sigmoid | 去掉模型最后的 Sigmoid |
| CrossEntropyLoss 学不动 | 模型最后加了 Softmax | 去掉 Softmax，直接输出 logits |
