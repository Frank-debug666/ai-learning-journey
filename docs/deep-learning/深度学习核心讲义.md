# 深度学习核心讲义：从数据契约到训练闭环

> 这份讲义不是 API 清单，而是一张解决深度学习问题的地图。  
> 核心目标：看到一个任务时，能够正确确定数据、模型输出、损失函数、训练流程和评估方式。

---

## 0. 先建立总框架

深度学习项目可以压缩为一个闭环：

```text
定义任务与指标
    ↓
准备数据：划分、清洗、转换、批量加载
    ↓
定义模型：输入 → 特征提取 → logits
    ↓
计算损失：logits 与 target 是否匹配
    ↓
反向传播：计算梯度
    ↓
优化器更新参数
    ↓
验证泛化能力并保存最优模型
    ↓
测试、部署、推理与持续监控
```

真正最容易出错的不是模型层数，而是以下四个契约：

1. **任务契约**：这是回归、二分类、多分类，还是生成任务？
2. **Shape 契约**：每一层输入输出的维度是否正确？
3. **dtype 契约**：特征和标签的数据类型是否满足要求？
4. **模式契约**：训练、评估、推理时，模型模式和梯度开关是否正确？

---

## 1. 任务决定最后一层与损失函数

先定任务，再设计模型最后一层。不要反过来。

| 任务 | 模型最后输出 | 标签 | 推荐损失 | 关键规则 |
|---|---|---|---|---|
| 回归 | `[B, 1]` 或与标签同 Shape | `float32`，与输出同 Shape | `MSELoss` / `SmoothL1Loss` | 输出与标签严格对齐 |
| 二分类 | `[B, 1]` 或 `[B]` logits | `float32`，0/1，与输出同 Shape | `BCEWithLogitsLoss` | 模型末尾不要加 Sigmoid |
| 多分类 | `[B, C]` logits | `[B]`，`long` 类别索引 | `CrossEntropyLoss` | 模型末尾不要加 Softmax |
| 多标签分类 | `[B, C]` logits | `[B, C]`，`float32`，每类 0/1 | `BCEWithLogitsLoss` | 每个类别独立判断 |
| 边界框回归 | `[B, 4]` | `[B, 4]`，同一种坐标格式 | `SmoothL1Loss` / IoU 类损失 | 坐标格式必须统一 |
| 序列生成 | `[B, T, V]`，训练时常展平为 `[B*T, V]` | `[B, T]`，常展平为 `[B*T]` | `CrossEntropyLoss` | 预测下一个 token |

### 最重要的两条避坑规则

```python
# 多分类：直接传 logits
loss_fn = nn.CrossEntropyLoss()
loss = loss_fn(logits, y)  # logits: [B, C], y: [B] long

# 二分类：直接传 logits
loss_fn = nn.BCEWithLogitsLoss()
loss = loss_fn(logits, y)  # logits 与 y Shape 相同，y 为 float
```

`CrossEntropyLoss` 内部已经完成 `LogSoftmax + NLLLoss`。  
`BCEWithLogitsLoss` 内部已经完成 `Sigmoid + BCELoss`。

训练时手动再加 Softmax 或 Sigmoid，会降低数值稳定性并造成重复计算。

## 2. Shape 是深度学习的主线

### 2.1 常用符号

| 符号 | 含义 |
|---|---|
| `B` | batch size |
| `C` | 通道数或类别数 |
| `H, W` | 图像高度、宽度 |
| `F` | 特征数 |
| `T` | 序列长度 |
| `D` | embedding / hidden 维度 |
| `V` | 词表大小 |

### 2.2 三条典型 Shape 路线

#### 表格 / MLP 多分类

```text
单样本 [F]
→ DataLoader [B, F]
→ Linear [B, hidden]
→ ReLU [B, hidden]
→ Linear [B, C] logits
→ CrossEntropyLoss(logits [B,C], target [B])
```

#### 图像 / CNN 多分类

```text
原始图片 [H, W, C]
→ ToTensor [C, H, W]
→ DataLoader [B, C, H, W]
→ Conv/Pool [B, C_out, H_out, W_out]
→ Flatten [B, C_out*H_out*W_out]
→ Linear [B, num_classes] logits
```

#### 文本 / Transformer

```text
token id [B, T]
→ Embedding [B, T, D]
→ Transformer [B, T, D]
→ 分类头 [B, C]
或生成头 [B, T, V]
```

### 2.3 常用层如何改变 Shape

| 操作 | Shape 变化 | 记忆方式 |
|---|---|---|
| `DataLoader` | `[F] → [B, F]` | 最前面增加 Batch |
| `Linear(in, out)` | `[..., in] → [..., out]` | 只改变最后一维 |
| `Conv2d` | `[B,Cin,H,W] → [B,Cout,Hout,Wout]` | 改通道，也可能改空间尺寸 |
| `MaxPool2d(2)` | `[B,C,H,W] → [B,C,H/2,W/2]` | 通道不变，空间减半 |
| `Flatten` | `[B,C,H,W] → [B,C*H*W]` | 保留 Batch，展平其余维 |
| `Embedding` | `[B,T] → [B,T,D]` | 每个 token id 变为向量 |
| `permute` | 维度顺序改变 | 数据不变，解释方式改变 |
| `unsqueeze` | 增加一个大小为 1 的维度 | 常用于补 Batch/Channel |
| `squeeze` | 删除大小为 1 的维度 | 小心误删 Batch |

卷积输出尺寸：

```text
H_out = floor((H + 2P - K) / S) + 1
W_out = floor((W + 2P - K) / S) + 1
```

### 2.4 广播机制

广播用于元素级运算。检查两个 Shape 时，**从右向左对齐**。每一维必须满足：

- 两个维度相等；
- 其中一个维度为 `1`；
- 其中一个维度不存在。

```text
(2, 3, 5) + (3, 1) → (2, 3, 5)   成功
(4, 3)    + (3, 3) → 报错          4 与 3 不兼容
```

警惕“能广播但语义错误”。例如回归输出 `[B, 1]` 与标签 `[B]` 可能发生意外广播，应主动统一 Shape。

## 3. dtype 与设备契约

| 数据 | 常用 dtype |
|---|---|
| 输入特征 `x` | `torch.float32` |
| 回归标签 | `torch.float32` |
| 二分类 / 多标签标签 | `torch.float32` |
| 多分类类别索引 | `torch.long` |
| token id / Embedding 输入 | `torch.long` |
| 布尔掩码 | `torch.bool` |

常用转换：

```python
x = x.float()
y_class = y_class.long()
x, y = x.to(device), y.to(device)

# GPU Tensor 转 NumPy
array = tensor.detach().cpu().numpy()
```

高频错误：

```text
mat1 and mat2 must have the same dtype
→ 输入 dtype 与模型参数 dtype 不一致

Expected object of scalar type Long
→ CrossEntropyLoss 标签或 Embedding 输入不是 long

Expected all tensors to be on the same device
→ 模型、输入、标签不在同一个设备
```

## 4. 模型：特征提取器 + 任务头

所有模型都可以先这样理解：

```text
输入
→ 特征提取器 Backbone
→ 高层表示
→ 任务头 Head
→ logits
```

| 数据类型 | 常见特征提取器 | 典型输入 |
|---|---|---|
| 表格 | MLP | `[B, F]` |
| 图像 | CNN / ViT | `[B, C, H, W]` |
| 序列 | RNN / LSTM / GRU / Transformer | `[B, T, D]` |

激活函数主要放在隐藏层：

| 场景 | 首选 |
|---|---|
| 常规 MLP / CNN | ReLU |
| ReLU 神经元坏死明显 | LeakyReLU |
| Transformer | GELU / SiLU 类激活 |
| RNN 门控 | Sigmoid + Tanh |

输出层是否加激活，要由损失函数和推理需求决定，而不是机械添加。

## 5. 训练闭环

### 5.1 batch、iteration、epoch

```text
batch_size：一次送入模型的样本数
iteration：处理一个 batch 并更新一次参数
epoch：完整遍历一次训练集
```

若训练集有 50,000 条数据，`batch_size=1024`：

```text
每个 epoch 的 iteration 数 = ceil(50000 / 1024) = 49
训练 3 个 epoch，共约 147 次参数更新
```

### 5.2 标准 PyTorch 训练模板

```python
for epoch in range(num_epochs):
    model.train()
    train_loss = 0.0
    train_correct = 0
    train_total = 0

    for x, y in train_loader:
        x, y = x.to(device), y.to(device)

        optimizer.zero_grad()
        logits = model(x)
        loss = loss_fn(logits, y)
        loss.backward()
        optimizer.step()

        batch_size = x.shape[0]
        train_loss += loss.item() * batch_size
        train_correct += (logits.argmax(dim=1) == y).sum().item()
        train_total += batch_size

    avg_loss = train_loss / train_total
    accuracy = train_correct / train_total
```

顺序口诀：

```text
清梯度 → 前向 → 算损失 → 反向 → 更新
```

### 5.3 为什么统计 Loss 时乘 batch_size

默认 loss 通常是当前 batch 的平均值。最后一个 batch 可能更小，因此应先还原为该批总损失，再除以总样本数：

```python
total_loss += loss.item() * batch_size
avg_loss = total_loss / total_num
```

## 6. 优化器、学习率与正则化

### 6.1 优化器选择

| 场景 | 推荐起点 |
|---|---|
| 入门项目 / 快速 baseline | Adam |
| Transformer / ViT / 大模型微调 | AdamW |
| CNN，追求更强泛化并愿意调参 | SGD + Momentum |
| 稀疏特征 | Adagrad |
| 小型二阶优化问题 | LBFGS |

最重要的超参数通常是学习率：

```text
学习率过大：Loss 震荡、发散、出现 NaN
学习率过小：下降很慢、长时间不收敛
```

### 6.2 正则化的目标

```text
降低训练误差 + 限制模型过度拟合训练集
```

| 方法 | 核心作用 |
|---|---|
| Weight Decay | 限制权重过大；AdamW 中使用更清晰 |
| Dropout | 训练时随机关闭部分神经元 |
| 数据增强 | 增加有效数据多样性 |
| Early Stopping | 验证集不再改善时停止 |
| Label Smoothing | 防止分类输出过度自信 |

判断过拟合的典型信号：

```text
训练 Loss 持续下降、训练指标持续上升
但验证 Loss 开始上升、验证指标停滞或下降
```

## 7. 训练、评估、推理必须分清

| 阶段 | 标签 | 梯度 | 参数更新 | 模型模式 | 目标 |
|---|---|---|---|---|---|
| 训练 | 需要 | 开启 | 是 | `model.train()` | 学习参数 |
| 评估 | 通常需要 | 关闭 | 否 | `model.eval()` | 测量泛化能力 |
| 推理 | 通常不需要 | 关闭 | 否 | `model.eval()` | 预测新数据 |

评估模板：

```python
model.eval()
with torch.no_grad():
    for x, y in val_loader:
        logits = model(x.to(device))
        # 计算 loss 与指标，不 backward，不 step
```

`model.eval()` 会改变 Dropout、BatchNorm 等层的行为；`torch.no_grad()` 关闭梯度记录。两者作用不同，评估时通常都要使用。

## 8. 从实验到企业工作流

一个可靠项目不仅要“训练出模型”，还要能复现、选择、部署和监控：

1. 明确业务目标、任务类型与成功指标。
2. 划分训练集、验证集、测试集，避免数据泄漏。
3. 记录数据版本、代码版本、随机种子和超参数。
4. 先建立简单 baseline，再增加模型复杂度。
5. 监控训练/验证 Loss、核心指标、学习率和资源占用。
6. 按验证集指标保存最优 checkpoint，并使用 Early Stopping。
7. 只在最终选择完成后使用测试集做一次公正评估。
8. 部署后监控延迟、错误率、数据漂移和业务指标。

指标必须与业务目标匹配：

| 情况 | 重点指标 |
|---|---|
| 类别均衡、错误代价接近 | Accuracy |
| 类别不均衡 | Precision、Recall、F1、PR-AUC |
| 漏报代价高 | Recall |
| 误报代价高 | Precision |
| 回归 | MAE、RMSE、R² |

## 9. Debug 的正确顺序

遇到报错或模型不学习时，按这个顺序检查：

```text
1. 任务是否定义正确？
2. 打印一个 batch 的 x.shape / y.shape / dtype / device
3. 打印 logits.shape，核对最后一层输出
4. 核对 loss 的输入、标签 Shape 与 dtype
5. 用极小数据集尝试过拟合，验证训练链路
6. 查看梯度是否存在、是否为 NaN/Inf
7. 查看学习率与 Loss 曲线
8. 再考虑换模型、优化器或增加复杂技巧
```

建议每次先做一个批次体检：

```python
x, y = next(iter(train_loader))
print("x:", x.shape, x.dtype, x.device)
print("y:", y.shape, y.dtype, y.device)

logits = model(x.to(device))
print("logits:", logits.shape, logits.dtype, logits.device)
```

## 10. 一页记忆版

```text
任务先行：
  回归 → 同 Shape float → MSE/SmoothL1
  二分类 → 同 Shape float logits → BCEWithLogits
  多分类 → [B,C] logits + [B] long → CrossEntropy

Shape 主线：
  DataLoader 加 B；Linear 改最后一维；CNN 用 BCHW；
  Embedding 把 [B,T] 变 [B,T,D]；Loss 前必须核对。

训练闭环：
  train → zero_grad → forward → loss → backward → step

评估推理：
  eval + no_grad；不 backward；不 step。

泛化：
  看验证集，不只看训练集；保存最优模型；防止数据泄漏。

Debug：
  先查任务、Shape、dtype、device、logits/target 契约，再换模型。
```

## 最后一句话

> 深度学习的核心不是记住所有 API，而是建立一条稳定的推理链：  
> **任务决定输出，输出决定损失；Shape 和 dtype 保证计算正确；训练闭环负责学习，验证与监控保证模型真的有用。**
