# 深度学习核心讲义：通俗版

> 这份笔记适合第一次系统理解深度学习时阅读。  
> 它和《深度学习核心讲义》讲的是同一套知识，但会少用术语，多用例子。

---

## 0. 先用一句话理解深度学习

深度学习就是：

```text
给模型很多带答案的例子
→ 模型先猜答案
→ 用损失函数判断猜得有多离谱
→ 根据错误调整模型参数
→ 重复很多次，让模型越猜越准
```

例如，我们想让模型判断一张图片是猫、狗还是兔子：

```text
输入图片
→ 模型输出三个分数：[猫 2.1，狗 0.3，兔子 -1.2]
→ 正确答案是“猫”
→ 损失函数计算这次猜得好不好
→ 优化器根据错误微调模型
```

训练模型和学生做题很像：

| 深度学习中的概念 | 类比 |
|---|---|
| 训练数据 | 练习题 |
| 标签 `target` | 标准答案 |
| 模型预测 | 学生写出的答案 |
| 损失 `loss` | 错得有多严重 |
| 反向传播 | 分析错误原因 |
| 优化器 | 根据错误修改解题方法 |
| 验证集 | 模拟考试 |
| 测试集 | 最终考试 |

**本章记忆：模型不是直接记答案，而是通过不断犯错和改正来学习。**

## 1. 做项目时，先判断任务类型

不要一上来就纠结模型有几层。第一步应该先问：

> 模型最后要回答什么问题？

### 1.1 常见任务

| 任务 | 模型要回答的问题 | 例子 |
|---|---|---|
| 回归 | 预测一个连续数值 | 明天气温是多少？房价是多少？ |
| 二分类 | 两个选项中选一个 | 是垃圾邮件吗？会违约吗？ |
| 多分类 | 多个类别中只选一个 | 图片是猫、狗还是兔子？ |
| 多标签分类 | 多个类别可以同时选中 | 图片里同时有“人、汽车、道路” |
| 序列生成 | 一个接一个地产生内容 | 生成文字、翻译、续写 |

### 1.2 为什么任务类型很重要

任务类型会直接决定：

1. 模型最后输出几个数；
2. 标签应该长什么样；
3. 应该使用哪个损失函数。

最常用的搭配如下：

| 任务 | 模型输出 | 标签 | 损失函数 |
|---|---|---|---|
| 回归 | 每个样本一个或多个预测值 | 浮点数，Shape 与输出相同 | `MSELoss` / `SmoothL1Loss` |
| 二分类 | 每个样本一个原始分数 | 0 或 1 的浮点数 | `BCEWithLogitsLoss` |
| 多分类 | 每个样本对每个类别各给一个分数 | 类别编号，类型为 `long` | `CrossEntropyLoss` |
| 多标签分类 | 每个样本对每个标签各给一个分数 | 多个 0/1，类型为浮点数 | `BCEWithLogitsLoss` |

### 1.3 什么是 logits

`logits` 只是模型输出的**原始分数**。

假设模型判断一张图属于三个类别：

```text
类别：猫    狗    兔子
分数：2.1  0.3  -1.2
```

这里的 `[2.1, 0.3, -1.2]` 就是 logits。分数最大的“猫”是模型当前最相信的答案。

训练多分类模型时：

```python
loss_fn = nn.CrossEntropyLoss()
loss = loss_fn(logits, y)
```

不要在模型末尾手动添加 `Softmax`，因为 `CrossEntropyLoss` 已经在内部完成了需要的计算。

训练二分类模型时：

```python
loss_fn = nn.BCEWithLogitsLoss()
loss = loss_fn(logits, y)
```

不要在模型末尾手动添加 `Sigmoid`，因为 `BCEWithLogitsLoss` 已经包含它。

**本章记忆：先确定模型要回答什么，再确定输出和损失函数。**

## 2. Shape：每份数据的“包装规格”

`Shape` 表示一个张量有几层、每层有多少个数据。

可以把它理解成商品包装：

```text
一盒有多少排 × 每排多少个 × 每个有多大
```

在深度学习中，模型的每一层都只接受特定包装规格的数据。规格不对，程序就会报错。

### 2.1 常用字母

| 字母 | 含义 | 例子 |
|---|---|---|
| `B` | 一批有多少个样本，Batch Size | 一次处理 32 张图片 |
| `C` | 图片通道数或类别数 | RGB 图片有 3 个通道 |
| `H, W` | 图片高度、宽度 | 224 × 224 |
| `F` | 一个样本有多少个特征 | 年龄、收入等 10 个特征 |
| `T` | 序列长度 | 一句话有 20 个 token |
| `D` | 每个 token 的向量长度 | 每个 token 用 768 个数表示 |

### 2.2 三种常见数据长什么样

#### 表格数据

一个人的数据有 10 个特征：

```text
单个人：[10]
一次处理 32 个人：[32, 10]，也可以写成 [B, F]
```

#### 图片数据

一次处理 32 张 RGB 图片，每张大小为 224 × 224：

```text
[32, 3, 224, 224]
  B   C    H    W
```

PyTorch 通常要求图片使用 `[B, C, H, W]` 顺序。

#### 文本数据

一次处理 32 句话，每句话保留 20 个 token：

```text
token 编号：[32, 20]，即 [B, T]
经过 Embedding 后：[32, 20, 768]，即 [B, T, D]
```

### 2.3 常用层会怎样改变 Shape

| 操作 | 通俗理解 | Shape 变化 |
|---|---|---|
| `DataLoader` | 把多个样本装成一批 | `[F] → [B, F]` |
| `Linear(in, out)` | 把每个样本最后一维换成新长度 | `[..., in] → [..., out]` |
| `Conv2d` | 从图片中提取局部特征 | `[B,Cin,H,W] → [B,Cout,Hout,Wout]` |
| `MaxPool2d(2)` | 缩小图片特征图 | 高和宽通常减半 |
| `Flatten` | 把多维特征摊平成一排 | `[B,C,H,W] → [B,C×H×W]` |
| `Embedding` | 把 token 编号变成向量 | `[B,T] → [B,T,D]` |
| `unsqueeze` | 增加一层大小为 1 的包装 | `[B] → [B,1]` |
| `squeeze` | 删除大小为 1 的那一层 | `[B,1] → [B]` |

### 2.4 一个 CNN Shape 变化例子

```text
输入图片             [32, 3, 28, 28]
经过卷积             [32, 16, 28, 28]
经过池化             [32, 16, 14, 14]
展平                 [32, 3136]
经过全连接层         [32, 10]
```

最后的 `[32, 10]` 表示：

- 这一批有 32 张图；
- 每张图都有 10 个类别分数。

### 2.5 广播：程序偷偷帮你复制数据

广播让不同 Shape 的张量也能进行加减乘除。

例如：

```text
[3, 4] + [4]
```

程序会把 `[4]` 想象成复制了 3 份，所以计算可以进行。

广播很方便，但也可能让错误悄悄发生：

```text
模型输出：[B, 1]
标签：   [B]
```

这两个 Shape 看起来都包含 `B` 个数，但它们方向不同，计算时可能被广播成意料之外的结果。因此，计算损失前应让输出和标签的 Shape 明确匹配。

调试时最实用的方法：

```python
print("x:", x.shape)
print("y:", y.shape)
print("logits:", logits.shape)
```

**本章记忆：Shape 就是数据的包装规格；进入每一层前，都要确保规格正确。**

## 3. dtype 和 device：数据的“材质”与“位置”

Shape 正确还不够，模型还会关心：

- `dtype`：数据是什么类型；
- `device`：数据放在 CPU 还是 GPU。

### 3.1 常用 dtype

| 数据 | 常用类型 |
|---|---|
| 模型输入特征 | `torch.float32` |
| 回归标签 | `torch.float32` |
| 二分类、多标签标签 | `torch.float32` |
| 多分类的类别编号 | `torch.long` |
| token 编号 | `torch.long` |
| 真或假的掩码 | `torch.bool` |

为什么多分类标签使用 `long`？

因为标签保存的是类别编号，例如：

```text
[2, 0, 1]
```

它表示第一个样本属于 2 号类别，第二个属于 0 号类别，第三个属于 1 号类别。

### 3.2 CPU 和 GPU 不能直接混算

模型和数据必须在同一个设备上：

```python
device = torch.device("cuda" if torch.cuda.is_available() else "cpu")

model = model.to(device)
x = x.to(device)
y = y.to(device)
```

GPU Tensor 转成 NumPy 前，需要先放回 CPU：

```python
array = tensor.detach().cpu().numpy()
```

### 3.3 常见报错翻译

| 报错关键词 | 通俗解释 | 优先检查 |
|---|---|---|
| `same dtype` | 数据类型不一致 | `x.dtype`、模型参数类型 |
| `Expected ... Long` | 本应传类别编号，却传了浮点数 | `y.long()` |
| `same device` | 有的数据在 CPU，有的在 GPU | 模型、`x`、`y` 的 `.device` |

**本章记忆：Shape 是包装规格，dtype 是材质，device 是存放位置，三者都必须匹配。**

## 4. 模型到底在做什么

一个模型可以简单拆成两部分：

```text
特征提取器 + 任务头
```

以猫狗分类为例：

```text
图片
→ 特征提取器发现耳朵、毛发、脸型等特征
→ 分类任务头根据这些特征给猫和狗打分
→ 输出 logits
```

不同数据通常使用不同的特征提取器：

| 数据 | 常见模型 |
|---|---|
| 表格 | MLP |
| 图片 | CNN、ViT |
| 文本和序列 | RNN、LSTM、GRU、Transformer |

隐藏层通常会使用激活函数。激活函数让模型可以学习复杂关系，否则堆很多层也很难表达复杂规律。

入门时可以这样选：

| 场景 | 常用激活函数 |
|---|---|
| 普通 MLP、CNN | `ReLU` |
| Transformer | `GELU` 或 `SiLU` |
| RNN 内部门控 | `Sigmoid` 和 `Tanh` |

注意：隐藏层使用什么激活函数，与输出层是否添加 `Softmax` 或 `Sigmoid` 是两件不同的事。

**本章记忆：模型前半部分找特征，最后的任务头负责给出答案。**

## 5. 模型是怎样训练的

### 5.1 batch、iteration、epoch

假设有 1000 道练习题：

- `batch_size=100`：每次做 100 道；
- `iteration`：做完一批题并改正一次；
- `epoch`：把全部 1000 道题做完一遍。

所以：

```text
每个 epoch 有 1000 ÷ 100 = 10 次 iteration
训练 5 个 epoch，大约会更新参数 50 次
```

### 5.2 一次参数更新发生了什么

标准顺序：

```text
清空旧梯度
→ 模型进行预测
→ 计算预测有多错
→ 反向分析每个参数的责任
→ 优化器更新参数
```

对应的 PyTorch 代码：

```python
optimizer.zero_grad()       # 1. 清空上一次留下的梯度
logits = model(x)           # 2. 前向传播：模型进行预测
loss = loss_fn(logits, y)   # 3. 计算错误程度
loss.backward()             # 4. 反向传播：计算梯度
optimizer.step()            # 5. 根据梯度更新参数
```

最容易忘记的是 `optimizer.zero_grad()`。PyTorch 默认会累加梯度，如果不清空，当前梯度会和上一次梯度叠加。

### 5.3 一个完整的多分类训练循环

```python
for epoch in range(num_epochs):
    model.train()
    total_loss = 0.0
    total_correct = 0
    total_num = 0

    for x, y in train_loader:
        x = x.to(device)
        y = y.to(device)

        optimizer.zero_grad()
        logits = model(x)
        loss = loss_fn(logits, y)
        loss.backward()
        optimizer.step()

        batch_size = x.shape[0]
        total_loss += loss.item() * batch_size
        total_correct += (logits.argmax(dim=1) == y).sum().item()
        total_num += batch_size

    avg_loss = total_loss / total_num
    accuracy = total_correct / total_num
```

为什么统计损失时要乘 `batch_size`？

因为 `loss.item()` 通常是这一批样本的平均损失，而最后一批的数据量可能更少。先还原每批的总损失，最后再除以总人数，得到的平均值才准确。

**本章记忆：清梯度 → 预测 → 算损失 → 反向传播 → 更新参数。**

## 6. 优化器、学习率与过拟合

### 6.1 优化器是怎样“改答案”的

模型内部有很多参数。反向传播会告诉我们：

```text
每个参数应该往哪个方向调整，才能让损失变小
```

优化器负责真正执行调整。

入门时可以这样选：

| 场景 | 推荐起点 |
|---|---|
| 普通入门项目 | `Adam` |
| Transformer、ViT、大模型微调 | `AdamW` |
| CNN，愿意认真调参并追求泛化 | `SGD + Momentum` |

### 6.2 学习率决定每次改多少

学习率可以理解成每次修改答案的幅度：

```text
学习率太大：每次改得太猛，可能来回震荡甚至越改越错
学习率太小：每次只改一点点，训练速度很慢
```

出现以下现象时，应优先检查学习率：

- Loss 剧烈震荡；
- Loss 不降反升；
- 出现 `NaN`；
- Loss 很久几乎不变化。

### 6.3 什么是过拟合

过拟合就像学生背熟了练习题答案，却不会做新题。

典型现象：

```text
训练集表现越来越好
验证集表现却停止提升，甚至越来越差
```

常见应对方法：

| 方法 | 通俗理解 |
|---|---|
| 更多训练数据 | 让模型见更多题型 |
| 数据增强 | 对现有数据做合理变化 |
| Weight Decay | 限制参数变得过于极端 |
| Dropout | 训练时随机暂时关闭部分神经元 |
| Early Stopping | 验证集不再进步就停止训练 |
| 简化模型 | 避免模型记忆能力过强 |

**本章记忆：优化器决定怎样改，学习率决定改多少；验证集用来发现模型是否只会背题。**

## 7. 训练、验证、测试和推理不要混淆

### 7.1 四个阶段分别做什么

| 阶段 | 用途 | 会更新参数吗 | 通常有标签吗 |
|---|---|---|---|
| 训练 | 让模型学习 | 会 | 有 |
| 验证 | 训练期间选择模型、调参数 | 不会 | 有 |
| 测试 | 最后公正评价一次 | 不会 | 有 |
| 推理 | 给真实新数据预测答案 | 不会 | 通常没有 |

可以继续用考试类比：

```text
训练集：平时练习题
验证集：用来调整学习方法的模拟考试
测试集：方法确定后的最终考试
推理：真正开始解决现实问题
```

不要反复看测试集再调模型，否则测试集也会变成另一份练习题，最终成绩就不再公正。

### 7.2 为什么评估时需要两个开关

```python
model.eval()
with torch.no_grad():
    logits = model(x)
```

它们作用不同：

- `model.eval()`：通知 Dropout、BatchNorm 等层，现在不是训练阶段；
- `torch.no_grad()`：不用记录梯度，节省内存和计算。

评估时两者通常都要使用。

**本章记忆：训练集负责学习，验证集负责选择，测试集负责最终评分，推理负责实际使用。**

## 8. 一个可靠项目的完整流程

一个模型能跑起来，不代表项目已经完成。比较可靠的顺序是：

1. 明确问题：到底要预测什么，成功标准是什么；
2. 收集并检查数据：数据是否足够、标签是否可信；
3. 划分训练集、验证集和测试集，避免数据泄漏；
4. 先建立一个简单 baseline，确认整条流程能工作；
5. 训练时记录 Loss、指标、超参数和模型版本；
6. 根据验证集选择模型，保存最佳 checkpoint；
7. 模型确定后，只用测试集进行最终评价；
8. 部署后监控速度、错误率、数据变化和业务效果。

### 8.1 什么是数据泄漏

数据泄漏是指模型在训练时，偷偷看到了本来不应该知道的信息。

例如：

- 测试集样本混进了训练集；
- 先用全部数据计算均值和方差，再划分数据集；
- 预测明天时，特征中意外包含了后天的信息。

数据泄漏会让离线成绩异常漂亮，但模型上线后表现很差。

### 8.2 指标要根据问题选择

| 情况 | 重点关注 |
|---|---|
| 类别比较均衡，错误代价相近 | Accuracy |
| 类别很不均衡 | Precision、Recall、F1、PR-AUC |
| 漏掉一个目标代价很高 | Recall |
| 错报一个目标代价很高 | Precision |
| 回归任务 | MAE、RMSE、R² |

例如疾病筛查通常更关心 Recall，因为漏掉真正患病的人代价较高。

**本章记忆：真正可靠的模型，不仅分数好，还要流程可复现、评估可信、上线可监控。**

## 9. 报错或模型不学习时，按什么顺序检查

不要一遇到问题就换模型。很多问题其实只是 Shape、dtype 或数据处理错误。

推荐按下面的顺序检查：

### 第一步：确认任务

```text
是回归、二分类、多分类，还是多标签分类？
最后一层输出数量正确吗？
损失函数选对了吗？
```

### 第二步：检查一个 batch

```python
x, y = next(iter(train_loader))
print("x:", x.shape, x.dtype, x.device)
print("y:", y.shape, y.dtype, y.device)
```

重点看：

- Shape 是否符合预期；
- 输入和标签类型是否正确；
- 数值是否异常；
- 类别编号是否在合法范围内。

### 第三步：检查模型输出

```python
logits = model(x.to(device))
print("logits:", logits.shape, logits.dtype, logits.device)
```

多分类任务通常应满足：

```text
logits: [B, C]
target: [B]，并且 dtype 为 long
```

### 第四步：尝试故意记住极小数据集

只取很少的样本反复训练。如果模型连这几个样本都无法学会，训练链路大概率有问题，例如：

- 标签错了；
- 没有调用 `optimizer.step()`；
- 梯度没有正确计算；
- 学习率不合适；
- 模型输出与损失函数不匹配。

### 第五步：再考虑复杂原因

确认基本流程没问题后，再检查：

- 梯度是否为 `NaN` 或 `Inf`；
- 学习率是否合适；
- 数据是否严重不均衡；
- 模型容量是否不足；
- 是否需要更好的优化器或正则化。

**本章记忆：先查任务、Shape、dtype、device 和训练步骤，最后才考虑换复杂模型。**

## 10. 从零完成一个多分类任务

假设目标是识别 10 类图片。

### 第一步：明确数据和答案

```text
输入 x：[B, 3, 224, 224]
标签 y：[B]，每个值是 0～9 的类别编号，dtype 为 long
```

### 第二步：设计模型输出

因为有 10 个类别，所以模型对每张图输出 10 个原始分数：

```text
logits：[B, 10]
```

### 第三步：选择损失函数

```python
loss_fn = nn.CrossEntropyLoss()
```

模型末尾不手动添加 `Softmax`。

### 第四步：训练

```text
model.train()
→ 清空梯度
→ 得到 logits
→ 计算 loss
→ loss.backward()
→ optimizer.step()
```

### 第五步：验证

```text
model.eval()
→ torch.no_grad()
→ 计算验证集 Loss 和 Accuracy
→ 保存验证表现最好的模型
```

### 第六步：最终测试与推理

模型和超参数全部确定后，用测试集做最终评价。部署后，对新图片进行推理：

```python
pred = logits.argmax(dim=1)
```

## 11. 一页速记

```text
深度学习在做什么：
  模型预测 → 计算错误 → 反向传播 → 优化器更新 → 重复

任务与损失：
  回归：输出和标签同 Shape、float → MSE / SmoothL1
  二分类：输出和标签同 Shape、float → BCEWithLogits
  多分类：logits [B,C]、标签 [B] long → CrossEntropy

Shape：
  B 是一批样本数
  图片通常是 [B,C,H,W]
  表格通常是 [B,F]
  文本 token 通常是 [B,T]
  计算 Loss 前必须核对 logits 和 target

训练顺序：
  zero_grad → forward → loss → backward → step

评估：
  model.eval() + torch.no_grad()
  不 backward，不 step

发现过拟合：
  训练集继续变好，但验证集开始变差

Debug：
  任务 → Shape → dtype → device → Loss → 梯度 → 学习率 → 模型
```

## 最后一句话

> 学习深度学习时，不需要一开始记住所有 API。  
> 先抓住这一条主线：**任务决定模型输出，模型输出决定损失函数；数据规格正确，训练循环才能让模型真正学会。**
