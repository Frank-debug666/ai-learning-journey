# 优化器 API 输入与输出分析

这一部分重点讲 PyTorch 里常见优化器 API：

```
torch.optim.SGD
torch.optim.Adam
torch.optim.AdamW
torch.optim.RMSprop
torch.optim.Adagrad
torch.optim.LBFGS
```

你可以把优化器 API 理解成：

> **输入：模型参数 + 更新规则参数**
>  **输出：一个 optimizer 对象，用来更新模型参数**

------

# 1. 所有优化器的共同 API 结构

大多数优化器都长这样：

```
optimizer = torch.optim.优化器名(
    params,
    lr=学习率,
    其他参数...
)
```

------

## 核心输入参数

| 参数           | 是否必须 | 作用                               |
| -------------- | -------- | ---------------------------------- |
| `params`       | 必须     | 告诉优化器要更新哪些模型参数       |
| `lr`           | 通常必须 | 学习率，控制每次参数更新的步子大小 |
| `weight_decay` | 可选     | 权重衰减，用来缓解过拟合           |
| 其他参数       | 可选     | 不同优化器有自己的控制项           |

------

## 最常见写法

```
optimizer = torch.optim.Adam(
    model.parameters(),
    lr=0.001
)
```

这里的输入含义是：

| 输入                 | 含义                                      |
| -------------------- | ----------------------------------------- |
| `model.parameters()` | 把模型中需要训练的参数交给优化器          |
| `lr=0.001`           | 每次更新参数时，步子大小大约由 0.001 控制 |

------

## 输出是什么？

输出是一个优化器对象：

```
optimizer
```

它本身不是预测结果，也不是损失值。

它的作用是：

```
管理参数更新过程
```

常用方法有：

| 方法                          | 作用               |
| ----------------------------- | ------------------ |
| `optimizer.zero_grad()`       | 清空上一轮梯度     |
| `loss.backward()`             | 反向传播，计算梯度 |
| `optimizer.step()`            | 根据梯度更新参数   |
| `optimizer.state_dict()`      | 保存优化器状态     |
| `optimizer.load_state_dict()` | 加载优化器状态     |

------

# 2. SGD API 输入输出分析 ⭐⭐⭐⭐⭐

## API 写法

```
optimizer = torch.optim.SGD(
    params,
    lr=0.01,
    momentum=0,
    weight_decay=0,
    dampening=0,
    nesterov=False
)
```

------

## 输入参数分析

| 参数           | 常用值               | 作用                         |
| -------------- | -------------------- | ---------------------------- |
| `params`       | `model.parameters()` | 要更新的模型参数             |
| `lr`           | `0.01` / `0.1`       | 学习率，决定每次更新步子大小 |
| `momentum`     | `0.9`                | 动量，让更新方向带有惯性     |
| `weight_decay` | `1e-4`               | 权重衰减，减少过拟合         |
| `dampening`    | 一般不用改           | 控制 momentum 的削弱程度     |
| `nesterov`     | `True` / `False`     | 是否使用 Nesterov 动量       |

------

## 参数影响

### 1. `lr`

学习率太大：

```
训练不稳定，loss 可能爆炸
```

学习率太小：

```
训练很慢，loss 降不下去
```

------

### 2. `momentum`

没有 momentum：

```
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.01
)
```

更新比较朴素，可能震荡。

加入 momentum：

```
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.01,
    momentum=0.9
)
```

效果：

```
减少震荡，加速收敛
```

------

### 3. `weight_decay`

```
optimizer = torch.optim.SGD(
    model.parameters(),
    lr=0.01,
    momentum=0.9,
    weight_decay=1e-4
)
```

作用：

```
限制参数变得过大，缓解过拟合
```

------

## 输出结果有什么作用？

SGD 输出的是 optimizer 对象：

```
optimizer = torch.optim.SGD(...)
```

它负责在训练中执行：

```
optimizer.step()
```

也就是：

```
根据当前梯度更新模型参数
```

------

## 适合场景

| 场景             | 是否推荐   |
| ---------------- | ---------- |
| CNN 图像分类     | 推荐       |
| ResNet 训练      | 推荐       |
| 入门理解优化器   | 强烈推荐   |
| Transformer 微调 | 一般不优先 |

------

# 3. Adam API 输入输出分析 ⭐⭐⭐⭐⭐

## API 写法

```
optimizer = torch.optim.Adam(
    params,
    lr=0.001,
    betas=(0.9, 0.999),
    eps=1e-8,
    weight_decay=0
)
```

------

## 输入参数分析

| 参数           | 常用值               | 作用                       |
| -------------- | -------------------- | -------------------------- |
| `params`       | `model.parameters()` | 要更新的模型参数           |
| `lr`           | `0.001`              | 学习率                     |
| `betas`        | `(0.9, 0.999)`       | 控制一阶、二阶动量         |
| `eps`          | `1e-8`               | 防止除以 0，提高数值稳定性 |
| `weight_decay` | `0` 或较小值         | 权重衰减                   |

------

## 参数影响

### 1. `lr`

Adam 常用学习率：

```
lr=1e-3
```

如果 loss 震荡明显，可以尝试：

```
lr=1e-4
```

------

### 2. `betas`

```
betas=(0.9, 0.999)
```

可以简单理解成：

| 参数    | 作用                 |
| ------- | -------------------- |
| `0.9`   | 记住过去梯度方向     |
| `0.999` | 记住过去梯度平方大小 |

一般初学不用改。

------

### 3. `eps`

```
eps=1e-8
```

作用：

```
防止分母为 0，让训练更稳定
```

通常不用改。

------

### 4. `weight_decay`

在 Adam 里也可以写：

```
weight_decay=0.01
```

但现代深度学习里，如果你确实需要 weight decay，更推荐用：

```
AdamW
```

------

## 输出结果有什么作用？

Adam 输出 optimizer 对象。

它在 `optimizer.step()` 时会综合考虑：

```
当前梯度
过去梯度方向
过去梯度平方大小
```

所以 Adam 更新参数时更“智能”，通常比 SGD 收敛更快。

------

## 适合场景

| 场景             | 是否推荐     |
| ---------------- | ------------ |
| 普通分类任务     | 推荐         |
| 普通回归任务     | 推荐         |
| MLP              | 推荐         |
| 快速跑通实验     | 推荐         |
| Transformer 微调 | 更推荐 AdamW |

------

# 4. AdamW API 输入输出分析 ⭐⭐⭐⭐⭐

## API 写法

```
optimizer = torch.optim.AdamW(
    params,
    lr=0.0001,
    betas=(0.9, 0.999),
    eps=1e-8,
    weight_decay=0.01
)
```

------

## 输入参数分析

| 参数           | 常用值               | 作用                 |
| -------------- | -------------------- | -------------------- |
| `params`       | `model.parameters()` | 要更新的参数         |
| `lr`           | `1e-4` / `1e-5`      | 学习率               |
| `betas`        | `(0.9, 0.999)`       | 控制动量估计         |
| `eps`          | `1e-8`               | 数值稳定             |
| `weight_decay` | `0.01`               | 权重衰减，防止过拟合 |

------

## 参数影响

### 1. `lr`

普通深度学习：

```
lr=1e-4
```

大模型 / BERT 微调：

```
lr=1e-5
```

学习率太大时：

```
预训练模型容易被破坏，效果变差
```

------

### 2. `weight_decay`

AdamW 的重点就是：

```
weight_decay=0.01
```

作用：

```
让参数不要变得过大，提高泛化能力
```

------

## Adam 和 AdamW 的关键区别

| 优化器 | weight decay 处理方式       |
| ------ | --------------------------- |
| Adam   | weight decay 和梯度更新耦合 |
| AdamW  | weight decay 和梯度更新解耦 |

一句话：

> AdamW 更适合现代深度学习模型，尤其是 Transformer。

------

## 输出结果有什么作用？

AdamW 输出 optimizer 对象。

它在训练中负责：

```
用 Adam 的方式更新参数
同时用更合理的方式做 weight decay
```

------

## 适合场景

| 场景           | 是否推荐 |
| -------------- | -------- |
| BERT 微调      | 强烈推荐 |
| GPT 类模型微调 | 强烈推荐 |
| ViT            | 强烈推荐 |
| Transformer    | 强烈推荐 |
| 普通小模型     | 也可以用 |

------

# 5. RMSprop API 输入输出分析 ⭐⭐⭐

## API 写法

```
optimizer = torch.optim.RMSprop(
    params,
    lr=0.001,
    alpha=0.99,
    eps=1e-8,
    weight_decay=0,
    momentum=0
)
```

------

## 输入参数分析

| 参数           | 常用值               | 作用                       |
| -------------- | -------------------- | -------------------------- |
| `params`       | `model.parameters()` | 要更新的参数               |
| `lr`           | `0.001`              | 学习率                     |
| `alpha`        | `0.99`               | 控制历史梯度平方的平滑程度 |
| `eps`          | `1e-8`               | 数值稳定                   |
| `weight_decay` | `0`                  | 权重衰减                   |
| `momentum`     | `0` 或 `0.9`         | 动量                       |

------

## 参数影响

### 1. `alpha`

```
alpha=0.99
```

表示：

```
非常重视历史梯度平方的平均情况
```

如果 alpha 小一点，优化器会更关注最近的梯度变化。

------

### 2. `momentum`

RMSprop 也可以加 momentum：

```
optimizer = torch.optim.RMSprop(
    model.parameters(),
    lr=0.001,
    momentum=0.9
)
```

作用：

```
进一步减少震荡，加速收敛
```

------

## 输出结果有什么作用？

RMSprop 输出 optimizer 对象。

它会根据近期梯度大小自动调整每个参数的学习率。

简单理解：

```
变化大的参数，步子小一点；
变化小的参数，步子大一点。
```

------

## 适合场景

| 场景           | 是否推荐    |
| -------------- | ----------- |
| RNN            | 可以        |
| 时间序列       | 可以        |
| 梯度不稳定任务 | 可以        |
| Transformer    | 一般不用    |
| 普通任务       | Adam 更常用 |

------

# 6. Adagrad API 输入输出分析 ⭐⭐⭐

## API 写法

```
optimizer = torch.optim.Adagrad(
    params,
    lr=0.01,
    lr_decay=0,
    weight_decay=0,
    eps=1e-10
)
```

------

## 输入参数分析

| 参数           | 常用值               | 作用         |
| -------------- | -------------------- | ------------ |
| `params`       | `model.parameters()` | 要更新的参数 |
| `lr`           | `0.01`               | 初始学习率   |
| `lr_decay`     | `0`                  | 学习率衰减   |
| `weight_decay` | `0`                  | 权重衰减     |
| `eps`          | `1e-10`              | 数值稳定     |

------

## 参数影响

### 1. `lr`

Adagrad 的学习率会随着训练逐渐变小。

所以初始学习率可以比 Adam 稍大一些：

```
lr=0.01
```

------

### 2. `lr_decay`

```
lr_decay=0
```

一般不用改。

如果设置更大，学习率下降更明显。

------

## 输出结果有什么作用？

Adagrad 输出 optimizer 对象。

它会自动调整每个参数的学习率：

```
经常出现的特征 → 学习率逐渐变小
很少出现的特征 → 保持相对更大的更新
```

------

## 适合场景

| 场景        | 是否推荐 |
| ----------- | -------- |
| 稀疏特征    | 推荐     |
| NLP 词向量  | 可以     |
| 推荐系统    | 可以     |
| 普通 CNN    | 不常用   |
| Transformer | 不常用   |

------

# 7. LBFGS API 输入输出分析 ⭐⭐

## API 写法

```
optimizer = torch.optim.LBFGS(
    params,
    lr=1,
    max_iter=20,
    tolerance_grad=1e-7,
    tolerance_change=1e-9
)
```

------

## 输入参数分析

| 参数               | 常用值               | 作用                       |
| ------------------ | -------------------- | -------------------------- |
| `params`           | `model.parameters()` | 要更新的参数               |
| `lr`               | `1`                  | 步长                       |
| `max_iter`         | `20`                 | 每次 step 内部最多迭代次数 |
| `tolerance_grad`   | `1e-7`               | 梯度足够小时停止           |
| `tolerance_change` | `1e-9`               | 损失变化足够小时停止       |

------

## 参数影响

### 1. `max_iter`

```
max_iter=20
```

意思是：

```
一次 optimizer.step() 里面，LBFGS 可能会多次评估模型和损失
```

所以它比普通优化器更特殊。

------

## LBFGS 最大特点：需要 closure

普通优化器：

```
loss.backward()
optimizer.step()
```

LBFGS：

```
def closure():
    optimizer.zero_grad()
    output = model(x)
    loss = criterion(output, y)
    loss.backward()
    return loss

optimizer.step(closure)
```

------

## 输出结果有什么作用？

LBFGS 输出 optimizer 对象。

但它更新参数时，需要反复调用 `closure()` 来重新计算：

```
前向传播
损失
反向传播
梯度
```

所以它适合小模型，不适合大规模深度学习。

------

## 适合场景

| 场景         | 是否推荐 |
| ------------ | -------- |
| 小模型       | 可以     |
| 风格迁移     | 可以     |
| 大模型训练   | 不推荐   |
| Transformer  | 不推荐   |
| 大数据集训练 | 不推荐   |

------

# 8. optimizer 常用方法输入输出分析

------

## 1. `optimizer.zero_grad()`

### API

```
optimizer.zero_grad()
```

------

### 输入是什么？

一般不需要输入参数。

也可以写：

```
optimizer.zero_grad(set_to_none=True)
```

------

### 参数影响

| 参数                | 作用                          |
| ------------------- | ----------------------------- |
| `set_to_none=False` | 把梯度清零                    |
| `set_to_none=True`  | 把梯度设为 None，可能更省内存 |

------

### 输出是什么？

没有返回重要结果。

它的作用是：

```
清空上一轮梯度，避免梯度累积
```

------

### 必须记住

训练循环里通常每轮都要写：

```
optimizer.zero_grad()
```

否则梯度会累积。

------

## 2. `loss.backward()`

### API

```
loss.backward()
```

------

### 输入是什么？

通常不需要输入。

它依赖的是：

```
loss 这个标量
```

------

### 输出是什么？

它不会直接返回预测结果。

它的作用是：

```
计算每个参数的梯度，并存到参数的 .grad 里面
```

比如：

```
for name, param in model.named_parameters():
    print(name, param.grad)
```

------

### 它和 optimizer 的关系

```
loss.backward()
optimizer.step()
```

含义是：

```
backward 负责算梯度
step 负责用梯度更新参数
```

------

## 3. `optimizer.step()`

### API

```
optimizer.step()
```

------

### 输入是什么？

普通优化器一般不需要输入。

但是 LBFGS 需要：

```
optimizer.step(closure)
```

------

### 输出是什么？

一般没有重要返回值。

它的核心作用是：

```
根据参数的 .grad 更新参数本身
```

------

### 参数更新前后对比

```
before = model.linear.weight.clone()

optimizer.step()

after = model.linear.weight
```

如果梯度正常，`after` 会和 `before` 不一样。

------

## 4. `optimizer.state_dict()`

### API

```
optimizer_state = optimizer.state_dict()
```

------

### 输入是什么？

无。

------

### 输出是什么？

输出一个字典，里面保存优化器状态。

比如 Adam 会保存：

```
一阶动量
二阶动量
当前 step 数
参数组信息
```

------

### 它有什么用？

保存训练断点：

```
torch.save({
    "model": model.state_dict(),
    "optimizer": optimizer.state_dict()
}, "checkpoint.pth")
```

这样下次可以继续训练，而不是从头训练。

------

## 5. `optimizer.load_state_dict()`

### API

```
optimizer.load_state_dict(checkpoint["optimizer"])
```

------

### 输入是什么？

输入之前保存的 optimizer 状态字典。

------

### 输出是什么？

一般没有重要返回值。

它的作用是：

```
恢复优化器之前的训练状态
```

这对 Adam / AdamW 很重要，因为它们内部有动量缓存。

------

# 9. 完整训练流程中的输入输出关系

可以把训练流程理解成这张表：

| 步骤 | API                     | 输入             | 输出 / 作用            |
| ---- | ----------------------- | ---------------- | ---------------------- |
| 1    | `model(x)`              | 输入数据 `x`     | 得到预测值 `logits`    |
| 2    | `criterion(logits, y)`  | 预测值和真实标签 | 得到损失 `loss`        |
| 3    | `optimizer.zero_grad()` | 无               | 清空旧梯度             |
| 4    | `loss.backward()`       | 损失值           | 计算梯度，写入 `.grad` |
| 5    | `optimizer.step()`      | 参数里的 `.grad` | 更新模型参数           |

------

# 10. 一段代码看懂所有输入输出

```
model = MyModel()

criterion = torch.nn.CrossEntropyLoss()

optimizer = torch.optim.AdamW(
    params=model.parameters(),   # 输入：模型参数
    lr=1e-4,                     # 输入：学习率
    weight_decay=0.01            # 输入：权重衰减
)

for x, y in dataloader:
    # x：输入数据
    # y：真实标签

    optimizer.zero_grad()
    # 输出作用：清空上一轮梯度

    logits = model(x)
    # 输入：x
    # 输出：logits，模型预测分数

    loss = criterion(logits, y)
    # 输入：预测 logits + 真实标签 y
    # 输出：loss，一个标量损失值

    loss.backward()
    # 输入：loss
    # 输出作用：计算梯度，存入每个参数的 .grad

    optimizer.step()
    # 输入：参数中的 .grad
    # 输出作用：更新模型参数
```

------

# 11. 优化器 API 选择速查表

| 优化器 API            | 主要输入参数                   | 输出对象作用               | 适合场景                |
| --------------------- | ------------------------------ | -------------------------- | ----------------------- |
| `torch.optim.SGD`     | `params`, `lr`, `momentum`     | 按梯度下降更新参数         | CNN、理解原理           |
| `torch.optim.Adam`    | `params`, `lr`, `betas`        | 自适应学习率更新参数       | 普通深度学习            |
| `torch.optim.AdamW`   | `params`, `lr`, `weight_decay` | 更合理地做 Adam + 权重衰减 | Transformer、大模型微调 |
| `torch.optim.RMSprop` | `params`, `lr`, `alpha`        | 根据近期梯度调整学习率     | RNN、时间序列           |
| `torch.optim.Adagrad` | `params`, `lr`                 | 稀疏特征自适应学习率       | NLP、推荐系统           |
| `torch.optim.LBFGS`   | `params`, `lr`, `max_iter`     | 二阶近似优化               | 小模型、风格迁移        |

------

# 12. 最后记忆版

```
优化器 API 的输入：
模型参数 + 学习率 + 优化器自己的控制参数。

优化器 API 的输出：
optimizer 对象。

optimizer 对象的作用：
不是预测，不是算 loss，而是负责更新模型参数。

zero_grad：
清空旧梯度。

backward：
根据 loss 计算梯度。

step：
根据梯度更新参数。

state_dict：
保存优化器状态。

load_state_dict：
恢复优化器状态。
```

最重要的一句话：

> **损失函数输出 loss，backward 根据 loss 算梯度，optimizer 根据梯度更新参数。**
